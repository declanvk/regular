Expand description

Backend for the DFA struct.

Separating the two types allows for an optimized representation of the transition and state storage, which can be the largest part in most DFAs.

See the DefaultDFAStorage for a sensible default backend.

Return the resulting state from transitioning from the current state given the sym symbol.Alphabet

Safety

This version of the transition function is exposed so that implementations can bypass underlying safety checks when specific variants are upheld.

  1. The current state must have been the result of a previous call to add_state.
  2. The sym symbol must be a valid symbol contained in the alphabet associated with this DFA.
